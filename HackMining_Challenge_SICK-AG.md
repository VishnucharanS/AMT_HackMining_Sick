Sensor Reliability in Mining Environments

Sensor-based, LiDAR, Coding, Signal Processing, Data Analysis

Task Description:

Outdoor sensors used in mining operations must function reliably despite harsh conditions. Dust, mud, and water can accumulate on sensor surfaces, degrading signal quality and causing missed detections of vehicles, equipment, or personnel. This creates both safety risks and operational challenges.

The goal of this challenge is to develop methods that allow sensors to detect when they are contaminated, understand how the contamination affects performance, and initiate an appropriate response.

Possible Approach and Expected Outcome:

Participants may analyze the provided sensor data and newly recorded scenarios to identify changes that indicate contamination such as dirt, dust, or moisture. Based on these observations, the system should determine when sensor performance becomes unreliable or safety‑critical.

A suitable outcome could be a small software module or script that:

· detects contamination from sensor readings,

· classifies the severity (e.g., “normal,” “reduced performance,” “critical”),

· triggers a maintenance signal.

The implementation is planned in Python using ROS. Prior experience with ROS is helpful but not required!

Provided Material:

· a SICK MultiScan LiDAR sensor including driver software

· a SICK VIsionary B-Two stereo camera including driver software

· the ability for participants to record their own test scenarios

· materials to contaminate the sensor (e.g., dust, mud, water)
